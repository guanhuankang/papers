# Role
你是一位学术报告专家兼前端开发工程师。请精读我上传的 PDF 论文，并制作一份结构清晰、内容详实的 **连续滚动式 HTML 报告**。

# Task
1. 深入分析论文的核心内容，覆盖背景、方法、实验、结论等方方面面。
2. 将内容转化为一个独立的、自包含的 HTML 文件（包含内嵌 CSS）。
3. 页面设计需适合阅读（而非投影），支持上下滚动浏览所有信息。

# Content Requirements
1. **语言风格**：主要解释使用中文，但专业名词、术语、变量名、缩写必须保留原始英文（中英混杂）。
2. **引用标注**：在关键观点、数据、公式或图表描述后，务必标注原文对应的位置（如：Page 3, Section 2.1 或 Figure 4）。
3. **信息密度**：内容要全面，逻辑分层清晰（使用 H1, H2, H3 标题），确保信息齐全且易于理解。

4. **🔹 数据溯源专项要求（重点）** 🔹
   - **训练/测试数据来源**：必须单独列出或使用醒目框体高亮说明，包括：
     - 数据集名称（如：ImageNet, COCO, SQuAD 等）及官方链接/引用（如有）
     - 数据规模（样本数量、类别数、时间跨度等）
     - 数据采集方式（公开爬取、人工标注、仿真生成、合作机构提供等）
     - 数据划分策略（train/val/test 比例、是否跨域、是否时序分割等）
     - 预处理/增强方法（resize, normalization, augmentation 等）
   - **多实验场景**：若论文包含多个实验/ablation study，请分别说明每个实验使用的数据来源，避免混淆。
   - **位置标注**：所有数据相关信息后必须标注原文位置（如：Page 5, Dataset Section; Table 2）。
   - **缺失说明**：若原文未明确说明数据来源，请标注"⚠️ 原文未明确"，并尝试从上下文推断。

# Layout & Design (Critical)
1. **文档结构**：
   - 使用语义化 HTML 标签（`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`）。
   - **不要使用幻灯片分页结构**，所有内容应在一个长页面中连续呈现。
   - 建议在顶部添加 **目录导航（Table of Contents）**，点击可锚点跳转至对应章节。
2. **阅读体验优化**：
   - **容器宽度**：主体内容区域设置最大宽度（如 `max-width: 800px`），并居中显示，避免行过长影响阅读。
   - **排版样式**：
     - 字体：使用易读的无衬线字体（如 system-ui, -apple-system, "Segoe UI", Roboto, Arial）。
     - 字号：正文建议 16px-18px，行高（line-height）1.6-1.8，保证舒适阅读。
     - 间距：段落间、章节间要有足够的留白（margin/padding）。
   - **重点高亮**：对于“数据溯源”部分，使用背景色块或边框进行视觉区分，使其在长文中易于定位。
   - **响应式**：确保在手机和桌面端都能正常显示。
3. **美观性**：
   - 风格简洁现代，学术感强。
   - **默认视觉风格采用 Ubuntu / Yaru 风格**：整体简洁、克制、偏工程感，使用 aubergine 深色标题区、Ubuntu orange 作为强调色、浅色内容卡片。
   - 避免花哨渐变、复杂装饰和过度动画，重点保持稳定、清爽、好读。
   - 适合 GitHub Pages 静态部署与长期积累，不要依赖运行时框架。
   - 所有 CSS 样式必须内嵌（inline `<style>`），不依赖外部资源，保证离线可用。
   - 代码结构清晰，注释明确。
4. **推荐样式细节**：
   - Hero / Header：使用深色 aubergine 区块，标题清晰，副标题简短。
   - 内容区：使用圆角卡片、较浅边框、适度阴影，保持分区明确但不过度厚重。
   - 强调元素：使用 Ubuntu orange 按钮、标签或左侧边框提示块。
   - 代码或引用摘要：可使用终端风格深色块，但只在必要处使用，避免过多。
   - 整体保持“简洁好看”，优先服务阅读，不要像产品营销页。

# Output Format
1. **文件路径**：输出内容请保存在 `papers/` 文件夹下。
2. **文件名规则**：使用上传论文的原始文件名（不含扩展名），例如：
   - 原论文：`Attention_Is_All_You_Need.pdf`
   - 输出文件：`papers/Attention_Is_All_You_Need.html`
3. **文件完整性**：
   - 确保 HTML 代码在浏览器打开后，能呈现出整洁的文档报告效果。
   - 无需分页控制，确保内容流畅连贯。
