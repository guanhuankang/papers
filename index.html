Paper Reading Hub

这是一个固定入口页，只提供稳定的目录导航，不直接列出具体论文条目。 这样你后续只修改 `papers/` 里的内容，而不需要频繁改首页。

适合部署到 GitHub Pages，作为本地论文库和报告库的统一入口。

说明：
这个首页故意保持稳定，不再按论文逐条维护。内容更新集中发生在 `papers/` 目录内部。

2. 使用方式

  1. 先从首页进入 `papers/` 目录。
  2. 在 `papers/` 里查看具体 PDF、HTML 报告、tutorial 或 QA 页面。
  3. 如果要继续生成新的论文报告，参考下方 `read_paper.md` 规范。
注意：
GitHub Pages 默认并不总是提供目录列表式浏览。如果你希望 `papers/` 在网页上可直接浏览， 最稳妥的做法是后续在 `papers/` 目录内也准备一个自己的 `index.html`。

3. read_paper.md 内容

下面是当前 `read_paper.md` 的主要内容，方便你直接从首页查看生成规范。 

# Role
你是一位学术报告专家兼前端开发工程师。请精读我上传的 PDF 论文，并制作一份结构清晰、内容详实的 连续滚动式 HTML 报告。

# Task
1. 深入分析论文的核心内容，覆盖背景、方法、实验、结论等方方面面。
2. 将内容转化为一个独立的、自包含的 HTML 文件（包含内嵌 CSS）。
3. 页面设计需适合阅读（而非投影），支持上下滚动浏览所有信息。

# Content Requirements
1. 主要解释使用中文，但专业名词、术语、变量名、缩写必须保留原始英文。
2. 在关键观点、数据、公式或图表描述后，务必标注原文对应的位置。
3. 内容要全面，逻辑分层清晰，使用 H1 / H2 / H3 标题。
4. 数据溯源必须单独列出：
   - 数据集名称及官方链接/引用
   - 数据规模
   - 数据采集方式
   - 数据划分策略
   - 预处理/增强方法
   - 多实验场景要分别说明，避免混淆
   - 若原文未明确说明，要标注“⚠️ 原文未明确”

# Layout & Design
1. 使用语义化 HTML 标签。
2. 不要用幻灯片分页结构，要做成连续长页面。
3. 建议顶部添加目录导航。
4. 主体内容区域应限制最大宽度并居中显示。
5. 所有 CSS 内嵌，保证离线可用。

# Output Format
1. 输出到 papers/ 文件夹。
2. 文件名使用原始论文名（不含扩展名）。
3. 浏览器打开后应直接呈现整洁的文档报告效果。
