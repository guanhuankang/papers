Marp Paper Hub

一个稳定、简洁的论文入口页。首页保持固定风格与导航，后续只需要更新 papers/ 目录中的 PDF、HTML 报告、 tutorial 或 QA 页面即可。

适合 GitHub Pages 部署，首页不承担内容维护，只承担入口与说明。

1. 入口

`papers/`

所有论文 PDF、HTML 解读报告、tutorial 和 QA 页面统一放在这里。 这是后续唯一需要持续更新的内容目录。

打开 `papers/`
`read_paper.md`

论文报告生成规范，包括页面结构、中文讲解方式、引用标注规则，以及重点的数据溯源要求。

打开 `read_paper.md`
设计原则：
首页固定，目录更新在 `papers/` 内部完成。这样部署到 GitHub Pages 后，后续新增论文时不必反复改首页。

3. 使用方式

  1. 部署后，从首页直接进入 papers/。
  2. 后续新增论文、报告、教程时，只修改 papers/ 内的内容。
  3. 如果需要查看报告生成要求，再返回首页打开 read_paper.md。
GitHub Pages 提醒：
如果你希望 `papers/` 在网页端始终可浏览，最稳妥的做法仍然是在 `papers/` 目录里准备一个自己的 `index.html`。 这样首页和子目录页都可以长期稳定不变。

4. 报告规范

下面保留 `read_paper.md` 的精简说明，方便快速回顾报告应该长什么样。

read_paper.md summary 
# Task
- 深入分析论文背景、方法、实验、结论
- 产出单个自包含 HTML 文件
- 页面应适合阅读，并支持连续滚动

# Key requirements
- 主要用中文解释，术语保留英文
- 关键观点后标注原文位置
- 单独写清楚数据来源、规模、划分、预处理
- 若原文未明确说明，要明确标注

# Output
- 文件输出到 papers/
- 文件名与原论文同名（不含扩展名）
- 所有 CSS 内嵌，适合离线查看
